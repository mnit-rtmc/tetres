Travel Time Reliability Estimation System (TeTRES)

Requirements

The following software is required to run the Tetres server:
  • python3
  • pip3
  • default-jre (not default-jre-headless)
  • python3-tkinter (can be installed by using sudo yum install python3-tkinter)
  • postgresql server
The client only needs Java to run the admin or user applications.

Installation and Running

Server instructions:
  • Download server.zip and extract to a directory.
  • Move server/tetres.conf.example to /etc/tetres/tetres.conf. Update the TICAS and PostgreSQL server details in this new file.
  • Run package_installer.sh to install the required libraries for python3. It will create a virtual environment, download and installs the required packages. You need to run only once.
  • To run the server, execute start_server.sh
Client instructions:
  • To run the user client, run tetresuser.jnlp with Java Web Start
  • To run the admin client, run tetresadmin.jnlp with Java Web Start

Client Configuration

When the user or admin is executed for the first time, it will automatically download the data and configurations. To access the following configurations, once TeTRES loads select Tools > Options.
  • PyTICAS Server URL - The system URL or IP on which PyTICAS server is running.
  • PyTICAS Server Port - The port on which PyTICAS server is running. (default 5000)
  • Data directory path - The directory containing the containing the configuration files (like filters, maps, ...) and temp data.
  • Log Level - Determines what the application logs to the console. This has no visible effect if the Java console is not enabled.
  • Map Provider - The source of the map. Choose TMC to use MnDOT's internal map tile server.

Data Files Download

If the user or admin client could not download the data directory, you can download the compressed files here. Extract the data folder anywhere, and in the Configurations window (Tools > Options) click browse and select the data directory.

Tested on Ubuntu 16.04, Fedora 28, and Windows 7.
